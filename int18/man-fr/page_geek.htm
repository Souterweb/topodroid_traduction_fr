PARAMÈTRES PERSONNALISÉS

Les paramètres personnalisés sont utilisés pour les fonctions ésotériques de TopoDroid.
Ils ne sont accessibles qu'au niveau du testeur T.
Cependant, leurs effets peuvent être visibles à un niveau inférieur (indiqué entre crochets).
Par défaut, les marqueurs sont désactivés.

Paramètres généraux

  • Retour simple : fermer les fenêtres d'un simple appui sur le bouton RETOUR
  • Jeux d'outils supplémentaires : pour installer des outils provenant d'autres jeux que le jeu d'outils de base de la cavité en cours.
  • Enregistrement des paquets : les paquets de données brutes sont copiés dans la base de données compilée.
  • Édition de croquis : active l'outil d'édition de croquis dans la fenêtre principale.



Paramètres des données

  • Mode plongée : permet d'obtenir des données en "mode plongée" : distance, azimut et profondeur.
  • Visées récentes : peuvent être surlignées en bleu et affichées de manière sélective.
  • Délai d'attente pour les visées récentes : temps après lequel une visée n'est plus "récente".
  • Extension fractionnée pour les visées de cheminement en coupe développée.
  • Permutation des stations pour les données de rétrovision DistoX
  • Direction et pendage du plan d'un ensemble de visées latérales.
  • Mesures des capteurs [A]
  • Bouclage (fermeture de boucle) [E]
  • Seuil de boucle : seuil utilisé dans la compensation sélective de la boucle [E].
  • Utilisation du compas/clino d'Android [A]
  • Télécommande du minuteur de visée (timer) [E] [10 s]
  • Télécommande du volume sonore de la minuterie (timer) [E] {10,100}. [50%]
  • Volume sonore de la télécommande du timer [E]
  • Visées de cheminement erronées [T]
  • Réaffectation des stations d'affichage
  • Renommer un groupe de visées d’habillage

 

Paramètres des visées latérales d'habillage

  • Classes des visées latérales : classification des visées latérales d'habillage en plusieurs catégories (Normales, X transverses, Horizontales et Verticales)
  • Coloration des visées latérales d'habillage [aucne]
  • Affichage des visées latérales d'habillage sous forme de points à leur extrémité [T].
  • Pente maximale des visées latérales sur le plan [80°] [A]
  • Mode d'affichage en pointillés des visées latérales : aucun, par pente, par azimut, "by th view". [Aucun]
  • Seuil de pente des visées latérales en pointillés : les visées latérales dont l'inclinaison est supérieure à ce seuil sont en pointillés. Ceci s'applique en mode d'affichage par pente, ou sur le plan en mode "by th view". Cela s'applique également à la coupe projetée en mode "by th view" pour les visées latérales qui font avec la direction de projection un angle inférieur à ce seuil . [50°].
  • Seuil d'azimut des visées latérales en pointillés : les visées latérales dont l'écart d'azimut est supérieur à ce seuil sont en pointillés. Ceci s'applique en mode d'affichage par azimut, ou pour une coupe développée en mode "by th view" [60°].
  • Seuil angulaire des visées latérales en pointillés : les visées latérales dont l'angle est inférieur à ce seuil sont mises en pointillés sur les sections transerses. [60°].
  • Couleur du tiret pour les visées d’habillage
  • Couleur du point pour les visées d’habillage C
  • Couleur de la dernière visée d’habillage

Paramètres de croquis

  • Décalage et mise à l'échelle d'un croquis.
  • Fractionnement / Fusion de croquis
  • Nombre de sauvegardes de croquis, conservées en boucle dans une liste [5] [A]
  • Intervalle minimum entre deux sauvegardes [60 s] [A]
  • Inclusion automatique des sections transverses dans les exportations [oui].
  • Stations sauvegardées colorées en orange dans les croquis.
  • Rafraîchissement des visées de cheminement uniquement : recalculer (et redessiner) la ligne polygonale médiane uniquement lorsqu'une visée de cheminement est téléchargée (en mode de téléchargement de données continu) [non T].
  • Transformation affine : active la transformation affine des croquis.
  • Niveaux de fond d'écran (Canvas) : aucun, par item, par type [aucun].
  • Ajustement de l'échelle du papier millimétré [0]
  • Sections transverses inclinées : autoriser ou non les sections obliques sur les vues en coupe [non]


L'actualisation par étapes du cheminement (Leg-only) n'affecte pas le mode de téléchargement des données à la demande. Il n'est pas non plus utilisé au niveau testeur [T], car, à ce niveau, il existe un rafraîchissement incrémentiel expérimental.

Si l'option sections obliques est activée, le plan des sections obliques dans les coupes peut présenter une inclinaison. Seules les inclinaisons multiples de 10 sont utilisées. Si les sections obliques ne sont pas activées, les lignes de section tracées supérieures à la valeur de HThreshold sont des sections verticales, sinon ce sont des sections horizontales.

Paramètres de lignes

  • Angle de coin minimal pour redresser les lignes (quatrième bouton de la boîte de dialogue d'édition des éléments de ligne) [45°].
  • Précision d'interpolation pour les segments cubiques de Bézier [1.0].
  • Seuil d'angle pour les segments cubiques de Bézier [20.0].
  • Distance maximale point/segment pour les points superflus de lissage [0,5 m].
  • Longueur maximale du segment pour le lissage [2,0 m].
  • Mémoire tampon d’élimination : tampon autour d'un segment de ligne à simplifier [10 pxl].
  • Actions d'accrochage des lignes/zones
  • Actions de lissage/redressement de segment
  • Action de redressement de ligne entière
  • Action de multisélection
  • Actions composites

 

Paramètres de l'appareil périphériqe
  • Délai de connexion : délai [en 1/10 de secondes] avant d'essayer de se connecter au DistoX [par défaut, aucun délai] [B]
  • Second DistoX : si l'on souhaite travailler avec deux appareils distincts (DistoX uniquement)
  • Pause acquisition des données : pause après chaque téléchargement de données [valeur par défaut 250 ms] [A]
  • Pause validation des données : délai d'attente pour vérifier la validation des données [valeur par défaut 500 ms] [A]
  • Pause laser : pause après avoir mis en marche le laser [valeur par défaut 1000 ms] [A]
  • Pause visée : pause après la commande de visée à distance [2000 ms] [A]
  • Vérification du firmware [oui] [A]
  • Mode BRIC4 : soit les données primaires uniquement, toutes les données, ou toutes les données sauf l'index. Avec toutes les données, TopoDroid utilise comme index de visée la valeur du BRIC4. [tous] [T]
  • Données nulles BRIC4 : enregistre ou non les visées de longueur nulle [no] [N]
  • Bug SAP5 bit16 : solution de contournement [T]

    Remarque. Si les délais après les commandes à distance "pause laser" et "pause visée" sont trop courts, les commandes peuvent échouer.

Paramètres d'Import-Export

  • Inclure les fichiers outils dans l'archive zip
  • Mode des données du relevé topo à l'importation (normal, plongée) [normal].
  • Exportation de csx au format cSurvey-transfer (nécessite cSurvey 1.20)
  • Stations : ajout automatique de stations aux fichiers th2
  • LRUD splay count : nombre de visées d'habillage utilisées pour LURD
